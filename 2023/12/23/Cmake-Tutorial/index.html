Exercise 1 Building a Basic Project

Note: Both add_executable and add_library doesn’t actually compile or link anything immediately. Instead, it informs CMake about the existence of an executable target and specifies the source files associated with that target.

Remark: CMake itself does not perform the compilation and linking; it generates build system files (e.g., Makefiles or project files) that are used by external build tools to perform these tasks.

To summarize, CMake’s role is to generate build system files based on your project configuration. The actual compilation and linking take place when you run the build tool, which is a separate step from running CMake.

The command $ cmake --build <path-to-build-directory> is one of build tools.

Exercise 1 - Creating a Library

First, we get into MathFunctions directory and update CMakeLists.txt

  • add_library(<libname> SHARED|STATIC|MODULE <src> ... <src>): Add a library target.

Next, we come back to the source directory ($ cd ..) and update the CMakeLists.txt.

  • add_subdirectory(src_dir): the add_subdirectory command is used to include the CMakeLists.txt file from another directory and build the targets defined in that subdirectory.
  • target_link_libraries(<target> <item> ... <item>): Link items to target.
  • target_include_directories(<target> PUBLIC|PRIVATE|INTERFACE <dir> ... <dir>): Specify directories for target to search while being compiled.

Difference between PUBLIC and PRIVATE

  1. PUBLIC:

    • Include directories specified with PUBLIC are added to the include directories for the target itself and any targets that link to it.
    • This means that if a target A includes directories with PUBLIC, and another target B links to A, B will also inherit those include directories.

  2. PRIVATE:

    • Include directories specified with PRIVATE are added only to the include directories for the target itself.
    • If a target B links to target A, it does not inherit the include directories specified with PRIVATE for A.
  • PROJECT_SOURCE_DIR: A variable that holds the absolute path of the root directory of the current CMake project. It is created by the last call to project().
  • PROJECT_BINARY_DIR: A variable that holds the absolute path of the build directory, where CMake generates build artifacts (or where we run the command cmake).

Right now we have configured the search (include) paths and the libraries to be linked by tutorial (our target executable file). We update tutorial.cxx so that the standard sqrt() function will be replaced with user-defined function.
